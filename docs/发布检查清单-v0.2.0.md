# PageCase v0.2.0 发布检查清单

## 本版要点

- 产品名：**PageCase**（安装包 `PageCase_0.2.0_win64_Setup.exe`）
- Changelog 仅保留 **0.2.0** + **0.1.0**；文案：「新增风格化文件选择器；优化了更新逻辑。」
- **GitHub：同一仓库改名** `PDF_Studio` → `PageCase`（不是新开仓）。改名后旧 URL 会 301 到新名，**v0.1.0 仍可更新**。
- 发 Release / `update.json` 一律按 **PageCase** 写；改名完成前若 raw/`releases` 仍只有旧名，先改名再上传，或临时让 `downloadUrl` 用旧名（改名后也会重定向）。

## 1. 版本号已对齐（发前再确认）

| 文件 | 期望 |
|------|------|
| `APP_VERSION.txt` | `0.2.0` |
| `src/appsettings.h` / `src/main.cpp` | `0.2.0` |
| `CMakeLists.txt` `project(... VERSION …)` | `0.2.0` |
| `resources/changelog.json` | 仅 0.2.0 + 0.1.0 |

## 2. `update.json`（决定 0.1.0 能否更新）

```json
{
  "version": "0.2.0",
  "downloadUrl": "https://github.com/MeowYewy/PageCase/releases/download/v0.2.0/PageCase_0.2.0_win64_Setup.exe",
  "silentInstallArgs": "/SILENT /SUPPRESSMSGBOXES /CLOSEAPPLICATIONS /RESTARTAPPLICATIONS",
  "releaseNotes": "新增风格化文件选择器；优化了更新逻辑。"
}
```

推荐发布顺序：

1. 仓库改名为 `MeowYewy/PageCase`（Settings → Rename）
2. 推送含本文件的 `main`
3. 创建 `v0.2.0` 并上传 Setup / portable
4. 用浏览器确认：
   - `https://raw.githubusercontent.com/MeowYewy/PageCase/main/resources/update.json`
   - 旧地址 `.../PDF_Studio/.../update.json` 也能打开（重定向）且 `version` 为 `0.2.0`

新客户端：先查 PageCase，失败再查 PDF_Studio。

## 3. 本机构建

```bat
cd /d D:\TechG\PageCase\desktop-qt
scripts\release.bat
```

产物应在 `dist\artifacts\`：

- `PageCase_0.2.0_win64_Setup.exe`
- `PageCase_0.2.0_win64_portable.zip`

注意：`*.bat` / `*.iss` 必须是 **CRLF**。`[Run]` 使用 `nowait postinstall`（不要用已废弃的 `skipifvery silent` / `skipifverysilent`，否则 ISCC 编译失败；且应用内 `/SILENT` 更新需要装完自动启动）。

## 4. GitHub Release

1. 推送含 `update.json` / `changelog.json` 的 `main`（**先于或同步于**发 Release，否则旧客户端仍看到旧 version）
2. 创建 tag `v0.2.0`
3. 上传上述两个 artifact；资产名必须与 `downloadUrl` 文件名一致
4. 用浏览器验证 raw `update.json` 的 `version` 为 `0.2.0`

## 5. 升级验收（重点）

| 场景 | 期望 |
|------|------|
| 已装 **PDF Studio 0.1.0** → 检查更新 | 检测到 0.2.0，能下载 |
| 点「安装」确认后 | 本进程有序退出；Inno `/SILENT` 进度条；装完自动启动 **PageCase** |
| 关于 / 版本号 | 显示 **0.2.0** / PageCase |

| 断网 / 坏包 | 显示失败（叉），可再次「检查更新」 |
| 重复点安装 | 不二次拉起安装包 |

## 6. 本版已做的异常防护（代码侧）

- Manifest 双 URL 回退（兼容 0.1.0 与改名后仓）
- 跟随安全重定向；超时；校验 HTTPS/URL
- 下载校验 MZ + 最小体积，拒绝 HTML/残缺包
- 安装仅启动一次；确认进程 PID 后再关窗退出
- Inno `CloseApplicationsFilter` 含 `PageCase.exe,PDFStudio.exe,ProjectP.exe`
